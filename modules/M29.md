## Spec

**`POST /mentor/ask`** — session auth, `requireActive()`
```json
{ "missionSlug": "deploy-to-vercel", "checkId": "build-passes",
  "question": "...", "errorText": "..." }
```
Zod: `question` 5–1000, `errorText` optional ≤ 5000 (truncated to 2000 in context).

**Order of operations — this exact order:**
1. Check the cap. **Before** calling Anthropic. Not after.
2. Build context (M28)
3. Call Anthropic
4. Persist `AiMessage` × 2 (user + assistant) with `tokensIn` / `tokensOut` **from the API response's usage object, not estimated**
5. Increment `aiCallsToday`
6. `track('mentor_asked', { missionSlug, checkId })` — **never the question text**
7. Return `{ answer }`

**Cap** — 25/day/user via `User.aiCallsToday` + `aiCallsResetAt`. Reset lazily: if `aiCallsResetAt` is yesterday, zero it and set to today. **No cron.**

At cap → 429: *"You've used today's mentor questions. They reset at midnight."* Not an error tone — a boundary. Add: *"Post in the feedback form if you're properly stuck."*

**Failure** — Anthropic down/timeout/rate-limited → `UPSTREAM_UNAVAILABLE` 503: *"The mentor is unavailable right now. Your work is unaffected."* **Do not increment the cap on a failed call.** Do not retry more than once. Log the real error server-side only.

**Timeout: 30s.** Then fail cleanly.

**`POST /cli/explain`** — same service, bearer auth, takes the last failing check from `checkResults` automatically. The learner types nothing.
