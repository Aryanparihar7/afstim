# M03 — Prisma schema, migration, health endpoint

Track: Foundation · Depends: M00 · Blocks: M04, M08, M12 · Agent time: ~2h

## Goal
The database exists, matches the spec exactly, and the live site can prove it's connected.

## Files you may touch
- `prisma/schema.prisma`
- `src/lib/prisma.ts`
- `src/app/api/v1/health/route.ts`
- `.env.example`
- `package.json` (prisma scripts only)

## Spec

**1.** Copy the schema from `PLAN.md` Section 6 **verbatim**. Do not add fields. Do not add models. Do not "improve" it. If something looks missing, **stop and ask** — it's probably deliberate.

Especially: there is no `Journey` model and no `Mission` model. Content lives in files. This is intentional and deletes an entire CMS from the build.

**2.** `src/lib/prisma.ts` — singleton with the dev hot-reload guard.

**3.** `npx prisma migrate dev --name init`

**4.** `GET /api/v1/health` → 
```json
{ "ok": true, "data": { "db": "ok", "commit": "<VERCEL_GIT_COMMIT_SHA or 'local'>" } }
```
DB check = `SELECT 1`. On failure → 503, `{ "ok": false, "error": { "code": "DB_UNAVAILABLE", "message": "Database is not reachable." } }`. **Never leak the connection error.**

**5.** `.env.example` with every key from `PLAN.md` Section 11, all values blank.

## Out of scope
Seeds. Any repository. Any service. Any UI.

## Done when
- [ ] `npx prisma migrate dev` runs clean
- [ ] `npx prisma studio` shows every table from Section 6 and no others
- [ ] tsc, lint, build pass

## Founder gate
1. `https://<yourdomain>/api/v1/health` → `{"ok":true,"data":{"db":"ok",...}}` ☐
2. Open Neon's dashboard. The tables are there. ☐
3. Break `DATABASE_URL` locally → health returns 503 with a plain sentence, **no stack trace**. ☐
