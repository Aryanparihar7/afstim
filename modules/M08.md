## Spec

Follow `docs/23_Authentication_Blueprint.md` ADRs. They are already decided — do not relitigate them.

- **Auth.js v5**, Credentials provider, **database session strategy** (not JWT — we need revocation, per ADR-008)
- Prisma adapter against the `Session` / `VerificationToken` models from M03
- **Argon2id** via `@node-rs/argon2`. Not bcrypt. Not scrypt. Not `argon2` (the C-binding one — it breaks on Vercel).
- `src/lib/env.ts` — Zod-validate every env var **at boot**. Missing `AUTH_SECRET` should crash the build, not fail at 2am.
- Session: 30 days, rolling refresh
- `authorize()` returns `null` for both "no such user" and "wrong password". **Identical timing, identical response.** Do not leak which one it was.

**Email/password only.** No Google. No GitHub. PRD §19 promises OAuth; ADR-011 defers it. The ADR wins. Do not add OAuth "since it's easy" — it is not in scope.

## Founder gate
Ask the agent two questions:
1. *"If someone steals the database, can they read passwords?"* — Correct: no, Argon2id is one-way.
2. *"Can a wrong email and a wrong password be told apart from outside?"* — Correct: no, same response, same timing.

Any hedging → reject.
