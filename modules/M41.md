# M41 — Backup + restore drill

Track: Launch · Depends: M40 · Blocks: launch · **Founder work, ~2h**

## Goal
You have restored the database once, on purpose, before you had to.

## Why this is the last gate

Everything else here is reversible. This isn't. If you lose the database, you lose every learner's progress, every reflection, and their trust — permanently. There is no `git revert` for that.

**A backup you have never restored is not a backup. It's a hope.**

## Do this

**1.** Neon → confirm point-in-time restore is on. Note the retention window. Write it down.

**2.** Create `docs/runbook-restore.md` **while you do the next step**, not after:
```
## Restoring the Afstim database
1. Neon console → project → Restore
2. Pick the timestamp
3. It creates a branch — it does NOT overwrite. Verify the branch first.
4. Copy the branch's connection string
5. Vercel → env vars → DATABASE_URL → paste → redeploy
6. Check /api/v1/health
7. Log in as yourself. Is your progress there?
Time this took me: ___ minutes
```

**3. The drill — actually do it.**
- Note your current mission
- Restore to a point-in-time from an hour ago, into a branch
- Point a **preview** deployment at the branch
- Log in. Is your data there?
- **Time it.**

**4.** Export a snapshot to your own machine:
```
pg_dump "$DATABASE_URL" > afstim-$(date +%F).sql
```
Do this weekly. A calendar reminder is a sufficient system at 20 users.

## Founder gate
1. You restored a database. With your hands. ☐
2. `docs/runbook-restore.md` exists, and you wrote it **while** doing it, so it's true. ☐
3. You know the number: restoring takes ___ minutes. ☐
4. A `.sql` file exists on your laptop, not only in the cloud. ☐

## Then

Every gate from M00 to M41 is ✅. The smoke test is green. The audit is clean. You have restored a database.

**Let 20 people in.**

Watch `/admin/events` every day. Watch one number:

> **Of the people who start mission 1, how many reach a live URL at mission 4?**

Under 50% → missions 1–4 are wrong. Fix the words. **Do not add features.** Adding features to escape a content problem is how the next eighteen months disappear.

Over 50% → you have something real. Now go find out what they'll pay for it.
