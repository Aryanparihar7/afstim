# M13 — Journey page

Track: Content · Depends: M12, M11, M02 · Blocks: M14 · Agent time: ~3h

## Goal
A learner sees the whole road, where they are on it, and what's next.

## Files you may touch
- `src/app/(app)/journey/[slug]/page.tsx`
- `src/components/afstim/mission-list-item.tsx`
- `src/features/progress/repositories/progress-repository.ts`
- `src/features/progress/services/progress-service.ts`

## Spec

**Header** — eyebrow `JOURNEY`, title, objective, `n of 10 verified` (mono).

**Mission list** — 10 rows. Each row: number (mono, `01`), title, one-line capability, duration, state.

States:
| State | Look | Clickable |
|---|---|---|
| `VERIFIED` | `✓` in `--pass`, title `--text-muted` | yes, read-only |
| `IN_PROGRESS` | title `--text`, subtle left border | yes |
| `AVAILABLE` | title `--text` | yes |
| `LOCKED` | title `--text-subtle`, `○` marker | **no**, and it says *why*: "Finish mission 3 first." |

**A locked row must always name its blocker.** Never a bare padlock. A learner should never wonder what to do.

**Unlock rule** — `AVAILABLE` when every `prerequisites` slug is `VERIFIED`. Mission 1 has no prerequisites. Optional missions never block anything.

**Empty state** — no progress yet: *"Mission 1 takes about 20 minutes. Start when you have a clear hour."* + `[Start mission 1]`.

## Out of scope
Mission content page (M14). Progress writes (M15 — this module reads only). Multiple journeys — one journey, hardcode the slug.

## Done when
- [ ] tsc, lint, build pass

- [ ] Zero N+1: **one** query for all progress, not one per mission
