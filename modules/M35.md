# M35 — Missions 1–3

Track: Authoring · Depends: M25, M00's log · **Founder work, ~4 days**

## Goal
The first three missions are real, and they came from your own errors.

## Why this is not agent work

An agent can format MDX. It cannot know that the thing that nearly made you quit was `gh auth login` silently failing behind a corporate proxy, or that you spent 40 minutes on a typo in a Neon connection string because the error said `P1001` and nothing else.

**That knowledge is the entire product.** It's the only thing here that Boot.dev, The Odin Project, Full Stack Open, and the ChatGPT tab don't have. If you let an agent invent these missions from general knowledge, you have built a tutorial — and a tutorial is exactly the thing Afstim exists to replace.

Open `docs/log/phase-1.md`. Start there.

## The three

| # | Slug | Capability |
|---|---|---|
| 01 | `set-up-your-machine` | The tools a real deploy needs, verified, not assumed |
| 02 | `create-and-commit` | Git is a safety net, not homework |
| 03 | `push-to-github` | Your code lives somewhere that isn't your laptop |

## How to write one

**Objective** — the result, not the topic.
- ✅ *"Get your code onto GitHub where a stranger could read it."*
- ❌ *"Learn about version control."*

**Prose** — 300–600 words. What you're doing, why it matters, what to type. **No theory that isn't needed in the next five minutes** (Product Bible, Principle 2).

**Checks** — the hard part.

> **Check outcomes, never names.**

- ✅ `command_succeeds: git log --oneline -1` — did you commit?
- ❌ `file_contains: "my-first-app"` — did you name it what I named it?

Every check must pass for **two learners with different code but working setups**. Before writing one, ask: *"could someone do this correctly and still fail?"* If yes, it's wrong.

**Hints** — a hint is what you'd actually say to a friend, out loud, at that moment.
- ✅ *"Read the first error. The rest are echoes of it."*
- ❌ *"Ensure your configuration is valid."*

Pull these from your log. Verbatim if possible.

**Reflection** — one question with no right answer, that only makes sense if they did the thing.
- ✅ *"If your laptop died right now, what would you lose?"*
- ❌ *"What is Git?"*

## The rule
**If you didn't personally hit it, don't write it.** Not one step, not one hint. The moment you write a step you haven't done, Afstim becomes a tutorial written by someone guessing — which is precisely the thing your learner already tried and quit.

## Founder gate
1. `npm run build` passes (M12's validator agrees your files are valid). ☐
2. **Do all three yourself, in a fresh folder, following only your own words.** ☐
3. **Give them to one real person.** Watch them. Say nothing. Write down every place they hesitate. ☐
4. That person reaches mission 3 without you helping. ☐

If step 3 hurts to watch — good. That's the most valuable four hours in this entire project. Every hesitation is a bug in your writing.
