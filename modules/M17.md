Track: Validation · Depends: M16 · Blocks: M18, M31 · Agent time: ~3h

`npx afstim login` works from a stranger's terminal.

## Spec

**Package** — name `afstim`, bin `afstim`, ESM, Node 20+.
**Dependencies: `commander`, `execa`, `picocolors`. Three. That is the entire allowlist.** No `chalk`, no `ora`, no `inquirer`, no `axios` — use `fetch`, it's built in.

**`afstim login`**
- Prompt: *"Paste your token from https://afstim.com/settings/cli:"* — **masked input**
- `GET /api/v1/cli/whoami` to validate
- Write `~/.afstim/config.json`, **mode 0600**: `{ token, apiUrl, email }`
- Success: `✓ Logged in as you@email.com`
- Bad token: *"That token isn't valid. Generate a new one at https://afstim.com/settings/cli"*

**`afstim status`** — current mission, `n of 10 verified`. Mono, quiet.

**`ui.ts`** — the terminal look:
```
▸ Mission 4 — Deploy to Vercel

  ✓ Your app builds without errors
  ✗ Your URL responds from the public internet
    → Open the URL on your phone with wifi off.

  1 of 3 passed. Run `npx afstim explain` if you're stuck.
```
Respect `NO_COLOR`. Never emoji.

**`api.ts`** — bearer auth, 10s timeout, and **network failure prints a human line, never a stack trace**: *"Can't reach Afstim. Check your connection and try again."*

## Out of scope
`check` (M19). `explain` (M31). Publishing to npm (M38). Self-update. Windows-specific paths — Mac/Linux first; if a learner is on Windows, WSL.

## Done when
- [ ] `npm run build` in `cli/`
- [ ] `node dist/index.js login` works against localhost
- [ ] Config file is 0600
- [ ] Ctrl-C mid-prompt exits cleanly, no half-written config

## Founder gate
1. In a scratch folder: `node <path>/cli/dist/index.js login`. Paste your token. `✓ Logged in`. ☐
2. `cat ~/.afstim/config.json` → your token. `ls -l` → `-rw-------`. ☐
3. Paste a garbage token → one plain sentence, no stack trace. ☐
4. Turn off wifi → `status` → one plain sentence. ☐
