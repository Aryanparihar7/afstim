# M16 — CLI token issue

Track: Validation · Depends: M11 · Blocks: M17 · Agent time: ~2h

## Goal
A learner can get a token that lets their terminal talk to Afstim as them.

## Files you may touch
- `src/app/(app)/settings/cli/page.tsx`
- `src/app/api/v1/cli/tokens/route.ts`
- `src/features/auth/services/cli-token-service.ts`
- `src/features/auth/repositories/cli-token-repository.ts`

## Spec

**`/settings/cli`**
- Explains in one sentence: *"This token lets `afstim` on your machine check your work."*
- `[Generate a token]`
- On generate: show the plaintext token **once**, in a `Terminal` block, with a copy button and the line *"Copy this now. You won't see it again."*
- Below: list of existing tokens — created date, last used, `[Revoke]`. Never the token itself.

**Token** — `afs_` + 32 random bytes hex, from `crypto.randomBytes`. **Not `Math.random`. Not `uuid`.**

**Storage** — `sha256(token)` in `CliToken.tokenHash`. **The plaintext is never persisted, never logged, never in an Event payload.**

**Verify** — hash the incoming bearer, look up by hash, reject if `revokedAt` is set, bump `lastUsedAt`.

**Limit** — 5 live tokens per user. Sixth → *"Revoke one first."*

## Out of scope
The CLI itself (M17). Any endpoint that uses the token (M18). OAuth device flow — a copy-paste token is correct for 20 users.
