# M05 — Event log service

Track: Foundation · Depends: M03, M04 · Blocks: M15, M34 · Agent time: ~1h

## Goal
Afstim can count things about itself. Ours, cheap, no third party.

## Spec

```ts
type EventType =
  | 'account_created' | 'email_verified'
  | 'mission_started' | 'check_run' | 'mission_verified'
  | 'reflection_saved' | 'mentor_asked' | 'submission_added'
  | 'journey_started' | 'journey_completed'

track(type: EventType, userId: string | null, payload?: Record<string, unknown>): Promise<void>
```

Rules:
- **Never throws.** Analytics failing must never break a learner's flow. Catch, log, swallow.
- **Never awaited in a request path** where it would add latency. Fire and forget.
- `payload` never contains: email, password, token, error text, file contents, URLs from user submissions.
- `EventType` is a closed union. Adding one requires a module that says so.

## Out of scope
Any UI. Any dashboard. Any third-party SDK — **no PostHog, no GA, no Sentry, no Mixpanel.** We have `Event` and a table in `/admin/events` (M34). That's it.
