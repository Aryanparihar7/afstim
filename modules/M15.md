## Spec

```ts
ensureEnrollment(userId, journeySlug): Enrollment
startMission(userId, missionSlug): MissionProgress   // idempotent
getProgress(userId, journeySlug): MissionProgressDTO[]
getCurrentMission(userId): { journeySlug, missionSlug } | null
isUnlocked(userId, missionSlug): boolean
```

**Rules**
- `startMission` on first page view. Idempotent — viewing twice must not reset anything or bump `attempts`.
- `getCurrentMission` = the lowest-order mission that is not `VERIFIED` and is unlocked. That single value drives the dashboard's one button.
- **DTOs only.** No Prisma model ever leaves the backend (doc 25's DTO rule).
- `track('mission_started')` on first start. `track('journey_started')` on first enrollment.
- Unlock = all `prerequisites` `VERIFIED`. Optional missions never gate anything.

**Concurrency** — two tabs opening the same mission must not create two rows. Use the `@@unique([userId, missionSlug])` constraint and upsert. Do not "check then insert".
