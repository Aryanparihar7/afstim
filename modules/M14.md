## Spec

**Layout — single column, 720px, centered. No sidebar. No panels. No split view.**

The docs specify a three-panel workspace (mission / editor / mentor). There is no editor — the learner is in their own VS Code on their own machine. So there is no workspace. This is a **reading page** with a checklist. Build it as one.

```
← Back to journey

eyebrow   MISSION 04 · 25 MIN
h1        Deploy to Vercel
lead      Objective sentence, --text-muted, 17px

[ MDX prose ]

────────────────────────
eyebrow   HOW YOU'LL KNOW IT WORKED
<Terminal> $ npx afstim check </Terminal>

CheckRow  ○ Your app builds without errors
CheckRow  ○ Your URL responds from the public internet
CheckRow  ○ HTTPS is working with a valid certificate

[ submissions slot — M23 ]
[ reflection slot — M24 ]
[ mentor slot — M30 ]
```

All checks render `pending` in this module. Nothing runs yet.

**MDX components** — style `h2`, `h3`, `p`, `ul`, `ol`, `code`, `pre`, `a`, `blockquote` to the tokens. `pre` uses the `Terminal` look. **No syntax highlighting library** — mono + `--text` is enough and saves a dependency.

Prose max width **68 characters**. Line height 1.65.
