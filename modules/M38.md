# M38 — Playwright smoke test + publish CLI

Track: Launch · Depends: M37 · Blocks: M40 · Agent time: ~3h

## Goal
The critical path can't break silently, and strangers can install the CLI.

## Files you may touch
- `e2e/smoke.spec.ts`
- `playwright.config.ts`
- `.github/workflows/ci.yml`
- `cli/package.json` (publish config)
- `cli/README.md`

## Spec

**One test. The critical path. Nothing else:**
```
register → verify (read token from DB) → mission 1 →
POST fake check results via API → reflection → verified → mission 2 unlocked
```

Not a suite. **One test that fails loudly if the product stops working.** You can't read code, so this is your safety net — it either goes green or it doesn't.

`ci.yml` — on push: `tsc --noEmit` → `lint` → `build` → `playwright test` against a test DB. Red = don't merge.

**Publish the CLI to npm:**
- `npm publish --access public` as `afstim`
- **Check the name is free first.** If taken → `@afstim/cli` and update every doc, every mission, and the landing page terminal block. Do this before writing anything else that says `npx afstim`.
- `cli/README.md`: install, login, check, explain. Ten lines.
- Version `0.1.0`

**Do not** add more tests. Unit tests you can't read are cargo cult. When something breaks in production, add a test for that thing. That's the whole strategy at 20 users.

## Out of scope
Coverage targets. Visual regression. Load testing. Testing every route.

## Done when
- [ ] The smoke test passes locally and in CI
- [ ] Deliberately breaking verification → the test goes red
- [ ] `npx afstim@latest --version` works from a fresh machine

## Founder gate
1. Push a commit → GitHub Actions goes green. ☐
2. Break `verification-service` on purpose, push → **CI goes red before Vercel deploys.** ☐
3. On a friend's laptop: `npx afstim@latest login` → works. ☐
