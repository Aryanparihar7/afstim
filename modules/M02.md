# M02 — UI primitives

Track: Foundation · Depends: M01 · Blocks: M06, M07, M09, M13 · Agent time: ~3h

## Goal
Seven components exist. Every screen in Afstim is built from these and nothing else.

## Spec

Run `npx shadcn@latest init`. Pull only: `button`, `input`, `textarea`, `card`, `badge`, `label`. Nothing else. Restyle them to the tokens.

**Button** — variants: `primary` (white bg, black text), `secondary` (transparent, `--border-strong` border), `ghost`. Sizes `sm | md`. Focus ring visible. **Never coloured.**

**Input / Textarea** — `--surface` bg, `--border` border, focus → `--border-strong`. Error state = border `--fail` **plus a message string** (never colour alone).

**Card** — `--surface` bg, 1px `--border`, `--r-lg`, padding 24.

**Badge** — mono, 11px, uppercase, `0.08em` tracking. Variants: `default`, `pass`, `fail`, `pending`.

**Eyebrow** — mono, 11px, 500, `0.08em`, uppercase, `--text-subtle`.

**CheckRow** — *the signature component of the product.*
```tsx
<CheckRow state="pending" | "pass" | "fail" label="..." hint?="..." />
```
- Mono, 13px, left-aligned
- Marker: `pending` → `○` in `--pending`; `pass` → `✓` in `--pass`; `fail` → `✗` in `--fail`
- On `pending` → `pass`, the marker cross-fades over `--t-base`. **This is the only celebratory motion in the product.**
- `fail` renders the hint underneath, indented, prefixed `→`, in `--text-muted`
- The marker is never the only signal — the label weight changes too (colour-blind safe)

**Terminal** — `--surface` bg, 1px `--border`, `--r-md`, mono 13px, `$` prompt in `--text-subtle`. Takes children. Static display, not interactive.

**`/dev/components`** — every component, every variant, on one page. Not linked from anywhere. Your visual regression check for the rest of the build.

## Out of scope
Modal, dropdown, tooltip, toast, table, tabs, avatar, skeleton. If a later module needs one, that module adds it.
