Track: Validation · Depends: M14, M04 · Blocks: M25 · Agent time: ~3h

## Goal
The learner tells us their live URL so we can go look at it.

- `src/features/validation/repositories/submission-repository.ts`

## Spec

**UI** — renders only if the mission's frontmatter has `submissions`:
```
eyebrow  YOUR LINKS
label    Paste your live URL
input    https://your-app.vercel.app
[Save]
```
Saved → shows the value + `[Change]`. Saving triggers a recheck (M21).

**API** — `POST /missions/[slug]/submission`, session auth, `requireActive()`.
- Zod: `kind` ∈ `public_url | repo_url`, `value` valid https URL, ≤ 500 chars
- `kind` must be declared in that mission's frontmatter → else 400
- **Run `url-guard` from M21 here too.** Reject at input, not just at fetch. Two gates.
- Upsert (one submission per user per mission per kind)
- `track('submission_added', { missionSlug, kind })` — **never the URL itself**

**Error copy** — plain, actionable:
- not https → *"That needs to start with https://"*
- localhost/private → *"That address only works on your own machine. Paste the public URL."*
- unparseable → *"That doesn't look like a URL."*

Verification (M22). Rendering results (M25). URL previews/screenshots.
