## Spec

**`/login`** — Email, Password, `[Log in]`, link to `/register`. Nothing else. No "remember me" (sessions are 30 days already). No social buttons.

**Failure copy — exactly this string:** *"That email and password don't match."* One message for every failure case. Never *"user not found"*, never *"incorrect password"*.

**Lockout** — add:
```prisma
model LoginAttempt {
  id        String   @id @default(cuid())
  email     String
  ip        String
  success   Boolean
  createdAt DateTime @default(now())
  @@index([email, createdAt])
  @@index([ip, createdAt])
}
```
- 5 failures for one email in 15 min → lock that email 15 min
- 20 failures from one IP in 15 min → lock that IP 15 min
- Locked: *"Too many attempts. Try again in 15 minutes."*
- **Success clears the counter.**
- Migration name: `add_login_attempt`

**Logout** — in the user menu. One click, no confirmation. Destroys the session server-side.

**Unverified email** → do not log in. Redirect `/verify` with the resend option.

## Out of scope
Password reset (not in v0.1). 2FA. Remember me. Sign out everywhere. CAPTCHA.

## Done when
- [ ] tsc, lint, build pass
- [ ] Wrong email and wrong password produce byte-identical responses
- [ ] 6th attempt → locked; correct password after a success → counter cleared

## Founder gate
1. Log in. Log out. Log in again. ☐
2. Wrong password 6 times → locked with a clear sentence and a time. ☐
3. Wait it out, log in correctly → works. ☐
4. Try logging in with an email that doesn't exist → **the exact same message** as a wrong password. ☐
