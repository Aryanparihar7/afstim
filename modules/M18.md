# M18 — GET /api/v1/cli/current

Track: Validation · Depends: M17, M15 · Blocks: M19 · Agent time: ~2h

## Goal
The CLI can ask "what am I supposed to be doing, and how do I know it worked?"

## Files you may touch
- `src/app/api/v1/cli/current/route.ts`
- `src/app/api/v1/cli/whoami/route.ts`
- `src/features/validation/services/check-plan-service.ts`
- `src/features/validation/dto/**`

## Spec

**`GET /cli/whoami`** → `{ ok: true, data: { email, name } }`

**`GET /cli/current`** → 
```json
{ "ok": true, "data": {
  "journeySlug": "ship-your-first-app",
  "missionSlug": "deploy-to-vercel",
  "missionTitle": "Deploy to Vercel",
  "missionOrder": 4,
  "checks": [
    { "id": "build-passes", "type": "command_succeeds", "label": "...",
      "run": "npm run build", "expectStdoutMatches": null, "hint": "..." }
  ]
}}
```

**Critical: `remote_*` checks are stripped from this response.** The CLI must never see them, never be told to run them, never be able to influence them. Server checks are the only thing a learner can't fake, so they never leave the server.

Auth: bearer token (M16). Invalid → 401 envelope.
No current mission → `{ ok: true, data: null }`. Not an error — a state.
`requireActive()`. `PENDING` → 403: *"Your access isn't open yet."*

**The `run` field is the only executable string that crosses this boundary, and it comes from a repo file — never from user input, never from the database.** Say this in a comment above the DTO so nobody later "improves" it.

## Out of scope
Running checks (M19). Results (M20). Server checks (M21).

## Done when
- [ ] tsc, lint, build pass
- [ ] Response contains **zero** `remote_*` checks — agent proves it
- [ ] No token → 401 envelope

## Founder gate
1. `curl -H "Authorization: Bearer afs_..." https://<domain>/api/v1/cli/current` → your current mission. ☐
2. Scan the JSON. **No `remote_` anywhere.** ☐
3. Same curl without the header → 401, plain. ☐
