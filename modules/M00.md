# M00 — Repo, first deploy, your log

Track: Foundation · Depends: — · Blocks: everything · **Founder work, ~1 week**

## Goal
You personally take an empty folder to a live HTTPS site on your own domain, and write down every error on the way.

## Why this is not agent work
Afstim teaches deployment. You have never deployed. Every mission, every check, every mentor hint in this product only exists on the other side of you having done this once. An agent cannot do it for you — not because it can't deploy, but because it can't be surprised, and the surprises are the curriculum.

## Steps

1. `npx create-next-app@latest afstim` — TypeScript, Tailwind, App Router, `src/` directory.
2. `git init`, first commit, push to a new GitHub repo.
3. Import the repo into Vercel. Deploy. Open the `*.vercel.app` URL.
4. Vercel → Settings → Domains → add your domain. Follow the DNS instructions at your registrar.
5. Wait for DNS. **This is the wait that teaches you mission 10.** Write down how it felt.
6. Create a Neon project. Copy the connection string.
7. Add `DATABASE_URL` to Vercel env vars **and** your local `.env`.
8. Change the homepage text. `git push`. Watch the live site change.

## The log — the actual deliverable

Create `docs/log/phase-1.md`. Every time something fails, paste it in:

```
## 2026-07-18 14:32 — Vercel build failed
Error: (paste it verbatim, all of it)
What I thought was wrong: ...
What was actually wrong: ...
What I typed to fix it: ...
How long it took: 40 min
How I felt: ready to throw the laptop
```

Verbatim. Not paraphrased. The exact string a learner will paste into Google is the exact string your mission hint must match.

## Also create

- `AGENTS.md` at repo root — copy from `PLAN.md` Section 13.
- Fill in `## Locked versions` with the actual installed versions from `package.json`. **From now on nothing gets upgraded.**

## Founder gate

1. `https://<yourdomain>` loads with a padlock. ☐
2. You changed a word, pushed, and the live site changed within 2 minutes. ☐
3. `AGENTS.md` exists with real version numbers in it. ☐
4. **`docs/log/phase-1.md` has at least 15 real errors.** ☐

If gate 4 is thin, either it went suspiciously smoothly or you didn't write it down. The log is worth more than the code.

## Post it
Day 1, publicly, wherever you post: *"I know how to code. I've never deployed anything. Starting today."* Your inexperience is an unfair advantage with about a three-month shelf life. Spend it now.
