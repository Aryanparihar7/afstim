# M25 — Live check UI

Track: Validation · Depends: M22, M23, M24 · Blocks: M27 · Agent time: ~3h
**This module completes the product. After it, Afstim works end to end.**

## Goal
The learner runs `afstim check` in their terminal and watches the website update.

## Files you may touch
- `src/components/afstim/mission-checks.tsx`
- `src/app/api/v1/missions/[slug]/state/route.ts`
- `src/app/(app)/mission/[slug]/page.tsx`

## Spec

**`GET /missions/[slug]/state`** → `evaluate()` output. Session auth.

**Polling** — not websockets, not SSE. A `setInterval`:
- Poll every **3s** while any check is `pending`
- Stop after **90s** → show `[Check again]`
- Stop immediately when verified
- **Stop when the tab is hidden** (`visibilitychange`). Do not poll a backgrounded tab — it's free money for our hosting bill and it's rude.

**Transitions** — when a row goes `pending → pass`, cross-fade over `--t-base`. Staggered by row. This is the **only** celebratory motion in the entire product. It must feel *quiet* — a thing settling into place, not a firework.

**On all-verified:**
```
✓ Mission verified.
Next: 05 — Add a database
[Start mission 5]
```
No confetti. No modal. No badge. No sound. The learner shipped something real; treating it like a mobile game cheapens it.

**States:**
| | |
|---|---|
| all pending | *"Run `npx afstim check` in your project folder."* + Terminal block |
| some fail | failing rows show hints. Below: *"Stuck? Ask the mentor."* (slot for M30) |
| pending remote | *"Still propagating. This can take a few hours."* + `[Check again]` |
| verified | the block above |

## Out of scope
Mentor (M30). Dashboard (M27). Websockets. Optimistic UI.

## Done when
- [ ] tsc, lint, build pass
- [ ] Backgrounding the tab stops polling — agent proves it in the network panel
- [ ] Polling stops after 90s, doesn't run forever
- [ ] Reduced motion → rows change with no transition

## Founder gate — **the big one**

Do a full mission as a learner. No shortcuts. No Prisma Studio.

1. Open mission 1 on one screen, terminal on the other. ☐
2. `npx afstim check`. **Watch the website update within 5 seconds.** ☐
3. Fail a check on purpose → the hint you wrote appears on the page. ☐
4. Fix it, re-run → the row goes green. ☐
5. Write the reflection → verified → mission 2 unlocks. ☐
6. Background the tab for 2 minutes → network panel shows **no polling**. ☐

**If step 2 feels good — if watching that row go green from your own terminal gives you something — the product works. That feeling is what you're selling. If it feels flat, stop and fix it here, before you build anything else.**
