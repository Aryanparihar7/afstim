# M04 — API contract: envelope, errors, Zod

Track: Foundation · Depends: M03 · Blocks: every API module · Agent time: ~2h

## Goal
Every API response in Afstim has the same shape, forever. No route invents its own.

## Files you may touch
- `src/lib/api/response.ts`
- `src/lib/api/errors.ts`
- `src/lib/api/handler.ts`
- `src/lib/api/validate.ts`
- `src/app/api/v1/health/route.ts` (refactor to use it)

## Spec

**1. `response.ts`**
```ts
type ApiSuccess<T> = { ok: true; data: T }
type ApiError = { ok: false; error: { code: string; message: string; field?: string } }
export function ok<T>(data: T, status = 200): Response
export function fail(code: string, message: string, status: number, field?: string): Response
```

**2. `errors.ts`** — a closed set. Nothing else may be thrown:
```
UNAUTHENTICATED 401  ·  FORBIDDEN 403      ·  NOT_FOUND 404
VALIDATION_FAILED 400 · RATE_LIMITED 429   ·  CONFLICT 409
UPSTREAM_UNAVAILABLE 503 · INTERNAL 500
```
Each has a **user-safe** default message. `INTERNAL` always says *"Something went wrong on our side. Try again in a moment."* — never the real error.

**3. `handler.ts`** — a wrapper that catches everything, logs the real error server-side, returns the safe envelope. **Every route handler is wrapped in this. No exceptions.**

**4. `validate.ts`** — `parseBody(req, schema)`. On failure → `VALIDATION_FAILED` with the field name and Zod's message. Never the full Zod dump.

**5.** Refactor health to use all of it.

## Rules this module locks in forever
- Route handlers are **thin**: parse → call service → format. No business logic, ever.
- **No route handler imports Prisma.** Only `features/*/repositories/**` may.
- Every input is Zod-validated. No exceptions.
- Errors never expose internals.
