# M31 — afstim explain

Track: AI · Depends: M29, M17 · Blocks: — · Agent time: ~2h

## Goal
Help without leaving the terminal.

## Files you may touch
- `cli/src/commands/explain.ts`
- `cli/src/index.ts`
- `cli/src/lib/ui.ts`

## Spec

```
$ npx afstim explain

▸ Mission 4 — Deploy to Vercel
  About: Your URL responds from the public internet

  Paste the error you're seeing (Ctrl-D when done):
  > _
```

Then:
```
  Vercel's build ran but your start command exited immediately.
  That usually means the build output went somewhere the runtime
  isn't looking.

  Next: open the deploy log, find the first red line, and check
  what directory it's serving from.

  22 questions left today.
```

**Flow**
1. `POST /api/v1/cli/explain` with the pasted text
2. Server picks the last failing check automatically — **the learner never says which mission**
3. Print the answer, wrapped at 76 chars, indented 2

**Flags:** `--error <file>` to read from a file instead of stdin. Nothing else.

**No failing check** → *"Nothing's failing right now. Run `npx afstim check` first."*
**At cap** → the same human sentence as the web.
**Mentor down** → *"The mentor is unavailable. Your checks still work."*

Same 3 dependencies. No new ones. Respect `NO_COLOR`.

## Out of scope
Interactive follow-ups. Conversation history in the terminal. A TUI. `--fix`. **Never anything that edits the learner's files.**
