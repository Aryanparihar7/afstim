# Purpose

This document defines the complete experience of a learner using ShipLab.

It describes how users discover the platform, become customers, complete learning journeys, and ultimately launch a production-ready application.

Every feature in ShipLab should improve this journey.

If a feature does not improve the user's journey, it should not be included in Version 1.

---

# Journey Philosophy

Users should never feel like they are taking an online course.

Instead, they should feel like they are building a real software product with an experienced mentor beside them.

The platform should continuously answer three questions:

* What am I building?
* Why am I building it?
* What should I do next?

Confusion should never be part of the learning experience.

---

# User Journey Overview

The complete journey consists of eight stages.

1. Discovery
2. Evaluation
3. Registration
4. Onboarding
5. First Success
6. Learning & Building
7. Deployment
8. Graduation

Each stage has a specific objective.

---

# Stage 1 — Discovery

## User Goal

Find a reliable way to learn how to build and launch real software.

## User Mindset

"I know some programming, but I don't know how to actually build something that people can use."

## Product Goal

Capture attention by promising a real transformation.

## Success Criteria

The visitor understands that ShipLab teaches how to build and launch production-ready software rather than simply teaching programming.

---

# Stage 2 — Evaluation

## User Goal

Determine whether ShipLab is worth trying.

## User Mindset

"Can this actually help me build something real?"

## Product Goal

Build trust.

The user should clearly understand:

* What they will build
* How the platform works
* Why it is different
* What outcome they can expect

## Success Criteria

The visitor decides to create an account.

---

# Stage 3 — Registration

## User Goal

Create an account quickly.

## Product Goal

Reduce friction.

Registration should require the minimum amount of information necessary to begin learning.

Progressive profiling should be preferred over asking everything at once.

## Success Criteria

The user reaches onboarding within a few minutes.

---

# Stage 4 — Onboarding

## User Goal

Receive a personalized learning path.

## Product Goal

Understand the learner.

During onboarding, ShipLab identifies:

* Existing programming knowledge
* Deployment experience
* Career goals
* Preferred learning pace

The onboarding experience should feel conversational rather than like filling out a long form.

## Success Criteria

The learner receives a personalized first journey.

---

# Stage 5 — First Success

This is the single most important stage in Version 1.

## User Goal

Experience a meaningful win as quickly as possible.

## Product Goal

Build confidence.

The learner should complete an achievable mission early and see a tangible result.

The first success should happen quickly enough to create momentum without sacrificing understanding.

## Success Criteria

The learner believes:

"I can actually do this."

---

# Stage 6 — Learning & Building

## User Goal

Progress through missions while building a real application.

## Product Goal

Maintain momentum.

Every mission follows the same structure:

Objective

↓

Learn

↓

Implement

↓

Validate

↓

Reflect

↓

Complete

The user should never spend long periods consuming passive content without building.

## Success Criteria

The learner consistently completes missions while understanding why each step matters.

---

# Stage 7 — Deployment

## User Goal

Launch a production-ready application.

## Product Goal

Remove fear around deployment.

Deployment should feel like the natural outcome of the learning journey rather than an intimidating final challenge.

The AI Mentor should provide guidance only when appropriate, encouraging understanding instead of replacing it.

## Success Criteria

The learner successfully launches a publicly accessible application.

---

# Stage 8 — Graduation

## User Goal

Recognize how much they have accomplished.

## Product Goal

Reinforce confidence.

Graduation should summarize the learner's journey and highlight the practical skills they have acquired.

The learner should leave with:

* A production-ready application
* Deployment confidence
* A repeatable workflow
* Motivation to continue building

## Success Criteria

The learner feels prepared to build and launch their next project independently.

---

# Emotional Journey

Throughout the product, the learner should move through these emotions:

Curiosity

↓

Excitement

↓

Focused Learning

↓

Confidence

↓

Achievement

↓

Ownership

↓

Independence

Whenever possible, the product should reduce frustration and replace uncertainty with clarity.

---

# Friction Points

The highest-risk moments for learner drop-off include:

* Registration
* Onboarding
* First deployment
* Repeated technical errors
* Subscription decision after the free trial

These stages require special attention in design and user testing.

---

# Journey Principles

Every stage of the journey should:

* Reduce uncertainty
* Encourage progress
* Provide immediate feedback
* Celebrate meaningful achievements
* Keep the learner focused on building

---

# Definition of Success

A successful user journey is one where a learner begins with basic programming knowledge and finishes with the confidence and ability to independently build and launch a production-ready application.

---

# Design Decisions

* Journey-first, not lesson-first.
* Practical implementation over passive consumption.
* Confidence is the primary outcome.
* AI acts as a mentor, not an autopilot.
* Every stage should naturally lead to the next.

---

# Known Risks

* Learners abandoning the journey before deployment.
* AI providing too much assistance.
* Missions becoming too difficult too quickly.
* Scope expansion reducing clarity.
