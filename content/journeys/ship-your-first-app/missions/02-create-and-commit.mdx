## Why this mission exists

A folder full of code with no commits is one bad `rm -rf` away from gone. Committing isn't paperwork — it's the one habit that makes every later mistake reversible.

## What you're going to do

1. Make a new folder for this project and open a terminal inside it.
2. Run `git init` to start tracking it with Git.
3. Create a README.md file with one line describing the project.
4. Run `git add .` to stage the file.
5. Run `git commit -m "Initial commit"` — the message should say what this commit does, not just "commit".
6. Run `git log` and confirm you can see the commit you just made.
